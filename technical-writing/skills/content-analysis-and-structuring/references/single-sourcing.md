# Single Sourcing: Building Modular Documentation

Kurt Ament의 *Single Sourcing: Building Modular Documentation*은 문서를 전체 단위의 '책'이 아닌 독립적이고 재사용 가능한 '모듈'들의 집합으로 설계하는 방법을 다룹니다.

## 핵심 개념 및 분석 방법

### 1. 단일 소싱 (Single Sourcing) 원칙
- 동일한 콘텐츠를 여러 곳(매뉴얼, 온라인 도움말, 튜토리얼 등)에서 중복 작성하지 않고, 하나의 소스에서 관리하여 다양한 포맷으로 출력하는 전략입니다.
- 이를 위해 콘텐츠는 특정 포맷이나 문맥에 강하게 종속되지 않아야 합니다.

### 2. 모듈형 문서 (Modular Documentation)
- 정보를 작고 논리적인 단위(모듈 또는 토픽)로 쪼개어 분석합니다.
- 모듈은 스스로 완결성을 가져야 하며(Self-contained), 다른 모듈과 쉽게 조립될 수 있어야 합니다.

### 3. 구조화의 기준
콘텐츠를 구조화할 때 다음과 같은 토픽 유형으로 분리하는 것을 권장합니다:
- **개념 (Concepts)**: 사용자가 작동 원리나 배경을 이해하는 데 필요한 정보.
- **태스크 (Tasks)**: 사용자가 목표를 달성하기 위해 따라야 하는 단계별 지침.
- **레퍼런스 (References)**: 필요할 때 찾아보는 빠른 참조 데이터(API 매개변수, 명령어 등).

## Agent 활용 지침
- 콘텐츠를 분석하고 `concept.md`, `task.md`, `reference.md`로 분류할 때, 이 단일 소싱 및 모듈화 원칙을 적용하여 특정 문서에 종속되지 않는 독립적인 모듈 형태로 정리하세요.

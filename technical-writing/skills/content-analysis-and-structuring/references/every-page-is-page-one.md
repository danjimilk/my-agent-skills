# Every Page is Page One (EPPO)

Mark Baker의 *Every Page is Page One*은 웹 시대의 기술 문서를 구조화하는 혁신적인 프레임워크를 제시합니다. 

## 핵심 분석 및 구조화 원칙

### 1. 무작위 진입 (Random Access)
- 독자는 문서의 1장부터 순서대로 읽지 않습니다. 대부분 검색 엔진이나 링크를 통해 필요한 내용이 있는 페이지로 직접 진입합니다.
- 따라서 **모든 페이지는 독자가 처음 만나는 '페이지 원(Page One)'**이라고 가정하고 콘텐츠를 분석 및 구조화해야 합니다.

### 2. 자체 완결성 (Self-contained Topics)
- 각 토픽이나 문서 모듈은 그 자체로 의미가 완결되어야 합니다.
- 앞선 페이지를 읽었다고 가정하거나, 이전 페이지에 의존하는 서술 방식(예: "앞서 설명했듯이...")을 피하고, 각 토픽이 특정 주제에 대해 완벽한 해답을 제공하도록 콘텐츠를 분리합니다.

### 3. 명확한 목적 (Specific Purpose)
- 각 토픽은 단 하나의 명확한 사용자 목적(질문이나 태스크)을 해결해야 합니다. 개념과 절차를 하나의 긴 글에 섞지 말고, 각각 독립적인 토픽으로 구조화합니다.

### 4. 풍부한 문맥 제공 (Providing Context via Links)
- 토픽 내에서 모든 선행 지식을 다 설명할 수는 없습니다.
- 대신 하이퍼링크와 관련 문서 링크(Related Links)를 적극적으로 배치하여, 정보가 부족한 독자가 스스로 문맥을 탐색할 수 있는 정보 아키텍처를 설계합니다.

## Agent 활용 지침
- `content-structure.md`를 설계할 때, 각 하위 문서(Tutorial, How-to 등)가 다른 문서에 강하게 의존하지 않는 자체 완결적인(Self-contained) 모듈이 되도록 콘텐츠를 배분하세요.

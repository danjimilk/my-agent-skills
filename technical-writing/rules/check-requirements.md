# 기술 문서 작업 시 필수 확인 사항

이 프로젝트(또는 플러그인) 내의 테크니컬 라이팅(Technical Writing) 관련 스킬을 사용할 때는, 작업을 시작하기 전에 반드시 프로젝트 루트의 `.docs/requirements/` 디렉터리에 있는 요구사항 파일들을 읽고 컨텍스트를 파악해야 합니다.

- `.docs/requirements/readers.md`: 어떤 독자를 대상으로 글을 써야 하는지 확인합니다.
- `.docs/requirements/target-product.md`: 다루어야 할 제품이나 서비스가 무엇인지 확인합니다.
- `.docs/requirements/required-docs.md`: 현재 작성하려는 문서가 전체 문서 세트 중 어디에 해당하며, 어떤 목표를 달성해야 하는지 확인합니다.

해당 파일들이 존재한다면 파일 내용을 참고하여 문서의 톤앤매너, 용어, 그리고 구성 방향을 결정하세요. 파일이 존재하지 않는다면 `requirements-analysis` 스킬을 먼저 실행하여 해당 파일들을 생성해야 합니다.
